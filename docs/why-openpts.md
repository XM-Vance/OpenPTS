# 为什么做 OpenPTS

> 写给电力交易行业的同行：为什么我们要把一套电力交易运营系统开源，以及它能帮你解决什么、不能解决什么。

## 一、一个真实的问题

如果你在一家售电公司、综合能源服务商、或者虚拟电厂运营商做技术，大概率听过这样的话：

> "我们得有一套自己的交易管理系统。"

然后你会面对这样的现实：

- **从零开发**，要覆盖客户档案、零售合同、负荷管理、结算、价格、绿电、调频、储能……几十个业务域。一个 5 人团队，估算下来至少 **6-12 个月**才能有个能用的雏形。
- **买商业产品**，动辄百万起步，而且和你当地的交易规则未必对得上，二次定制还要再付一笔。
- **找外包**，交付质量参差不齐，拿到手往往是个改不动的黑盒。

而这其中真正有差异化价值的，其实只是**算法和规则**（你的预测模型、你的报价策略、你所在省份的交易规则）。剩下 70%-80% 的工作——鉴权、权限、多租户、客户/合同/结算的 CRUD、审计、文档解析——是**每个电力交易系统都要做、但谁做都一样**的通用底座。

这 70% 的重复劳动，就是 OpenPTS 想消灭的东西。

## 二、OpenPTS 是什么

一句话：**一个能跑的、覆盖电力交易运营全流程的开源骨架，算法和规则留给你自己接。**

它不是一个 demo，也不是一个玩具。它来自一套真实在生产环境运行的系统，剥离了算法内核和具体省份规则后，把通用的运营框架开放出来：

- **后端**：Go 网关，300+ API，完整的 JWT 鉴权、RBAC 权限、多租户隔离、操作审计、定时任务
- **前端**：Next.js 15，30+ 业务页面，客户/合同/结算/价格/负荷等全流程
- **数据库**：83 对迁移，~190 张表，覆盖电力交易的标准数据模型
- **文档解析**：电费单/合同 OCR → 结构化入库的完整管线

你 clone 下来，`bash scripts/demo.sh`，两分钟就能看到一个带数据、带预测曲线的完整系统在跑。

## 三、它能帮你省什么

把 OpenPTS 当起点，你的工作从"造轮子"变成"装引擎"：

| 你本来要做的 | OpenPTS 已经做了 | 你只需专注 |
|---|---|---|
| 鉴权/权限/多租户架构 | ✅ 完整 | — |
| 客户/合同/结算 CRUD | ✅ 完整 | 业务定制 |
| 数据库建模 | ✅ 标准模型 | 你当地的表结构差异 |
| 负荷/价格预测 | 🔌 留了接口 | **你的预测算法** |
| 竞价/报价策略 | 🔌 可扩展 | **你的策略逻辑** |
| 交易规则 | 📋 通用规则表 | **你省份的规则数据** |

保守估计，用 OpenPTS 做起点，一个熟悉 Go + React 的开发者，**1-2 周就能接入自己的算法和规则，跑起一套针对你所在市场的系统**。相比从零开始，省下的不是几天，是几个月。

## 四、它不是什么（诚实的边界）

为了避免误导，把局限也说清楚：

- **不含预测算法**：负荷/价格预测端点默认返回 501（演示模式返回合成数据）。你的预测模型（相似日、时序、机器学习）需要自己接——但接口契约和数据 DTO 都定义好了，接入点很清晰。
- **不含竞价/出清算法**：竞价策略、市场出清、潮流计算这些没有。框架预留了扩展空间，但实现是你的。
- **不含具体省份规则数据**：规则表结构是通用的（key/value），但里面是空的。你所在省份的交易规则、电价参数需要你自己填。
- **不是开箱即用的成品**：它是一个**起点**，不是终点。你需要有开发能力（或团队）来接入算法、定制业务、适配本地规则。

换句话说：**OpenPTS 给你的是一副骨架和肌肉，心脏（算法）和血液（规则数据）要你自己装。** 但比起从一具完整的身体开始造，这已经省了大部分力气。

## 五、为什么开源

电力行业有个特点：**业务 Know-How 是核心壁垒，但运营系统的工程实现不是。**

每家公司的预测模型、报价策略、客户关系都是商业机密，理应保密。但"怎么设计一个支持多租户的客户档案系统""结算流程的表结构怎么建""RBAC 权限怎么和多租户结合"——这些是**全行业共享的工程问题**，各自闭门造车是巨大的浪费。

把通用底座开源，让整个行业站在同一个起跑线上，把竞争拉回到真正有差异的地方（谁的算法更准、谁的策略更优、谁更懂本地市场）。这对行业是好事，对每个从业者也是好事。

这也是 OpenPTS 的协作边界：**欢迎贡献通用能力**（业务 CRUD、架构优化、文档、测试），但**算法和规则各留各的**。

## 六、适合谁用

- **售电公司 / 综合能源服务商**：想自建交易系统但不想从零开始的技术团队
- **虚拟电厂 / 储能运营商**：需要一个能管客户、管结算、管调度的运营底座
- **电力交易技术开发者**：想研究一套真实的电力交易系统是怎么设计的
- **高校 / 研究机构**：需要一个真实业务背景的平台来验证算法（把你的预测模型接进来就能跑）

## 七、怎么开始

```bash
git clone https://github.com/XM-Vance/OpenPTS.git
cd OpenPTS
bash scripts/demo.sh     # 两分钟体验完整系统
```

然后看 README 的「算法接入指引」，找到预测端点的接入点，把你的算法接进去。

如果这个项目对你有帮助，欢迎 Star ⭐、提 Issue、提 PR。更欢迎在电力技术社区分享给你觉得需要它的同行——OpenPTS 的价值，取决于有多少人能用起来。
